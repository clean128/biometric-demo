Auth Face - Biometric Authentication API

API Overview
Auth Face is a biometric authentication microservice designed to provide secure user login and identity verification through facial recognition.

The API delivers a complete facial authentication flow, including face capture, liveness detection, and anti-fraud mechanisms, ensuring that only real and authorized users can access protected systems.

Built for server-to-server integration, Auth Face allows platforms to start a biometric session, redirect users to a secure face capture interface, and later retrieve the authentication result using a client-provided state reference (client_state).

For privacy and security, Auth Face does not store facial images. Only irreversible facial embeddings (mathematical representations of biometric features) are processed and stored, significantly reducing the risk of personal data exposure and simplifying compliance with national and international data protection regulations.

Key features include facial login and identity verification, liveness detection to prevent spoofing attempts (such as photos, videos, or replays), biometric conflict detection, and session-based authentication with secure redirects. The architecture is multi-tenant, scalable, and suitable for onboarding flows, recurring logins, and high-risk actions.

Auth Face is ideal for applications that require strong authentication, effective fraud prevention, and a frictionless passwordless user experience.

Client calls a biometric flow endpoint
API returns biometry_url
Client redirects the end user
User completes facial capture (with liveness detection)
User is redirected to: redirect_url?client_state={UUID}
Client retrieves the final result

Notes
success indicates that the API request was successfully processed
(it does not represent the biometric result).
state corresponds to the client_state provided when the session was created.
flow indicates the biometric operation type.
status reflects the current biometric session state
(pending, in_progress, success, fail, expired).
score represents the biometric confidence score (when applicable).
Biometric internal identifiers (such as subject references) are never exposed in this endpoint.
reason is present only when the session fails.
Clients should poll this endpoint until status is no longer pending or in_progress.
